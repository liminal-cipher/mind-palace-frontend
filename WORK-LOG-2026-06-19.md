## ① 3D 엔진 정교화 (memory-walk)

### 가구 이름 인식 (가구 N 제거)
- 이름 없는 GLB(loft·room_paris 등 "Cube"/"Object_4")는 가구 종류를 못 잡아 "가구 1·2…"로 표시되던 문제.
- **사전 보강**(duvet/locker/board/fireplace 등) + **기하학 폴백 분류기**(`geomGuess`): 바운딩박스 크기·바닥높이로
  소파/테이블/의자/수납장/러그 등 추정. 중복은 "의자 2"식. 구조물·천장 부착물은 제외.

### 카메라 가려짐 방지 (삼각측량)
- 마커로 이동(`goTo`) 시 카메라가 다른 가구·벽에 가려지던 문제(특히 modern_apartment_interior).
- `clearViewPos`: 마커 중심으로 **N(28)방향 레이캐스트**해 "열린 거리"를 재고, 가장 트인 방향에 카메라 배치
  (점수=열린거리+방중심향함+거리). 최종 `losClear`로 실제 시야 재확인. 폴백=정면 1.7m.
- 제시된 "레이캐스트+삼각측량" 식 검증 완료(오류 없음).

### 조명 (검은 void 해소)
- 조명 안 구운 GLB가 검게 죽던 문제 → **AmbientLight 0.42 + HemisphereLight 0.62** 추가(전역광).
  밝은 baked 방(modern_bedroom 등)은 그대로 유지됨을 라이브 확인.

### 동선(핫스팟 순서) 버그
- `designRoute` 정렬 비교자가 **비추이적**(섹터 안=깊이순, 밖=각도순)이라 정렬이 뒤섞이던 버그.
- 각도를 SECTOR(0.20rad) **버킷으로 양자화** → (버킷, 깊이) 전순서로 정렬해 좌→우 부채꼴 스윕이 일관되게.

### ②c 방 미리보기 캡처
- 중심→벽 정면 → **대각선 4방향 + 모서리로 물러서기**로 구도 개선(깊이·가구 프레이밍).
- 라이브 검증: 4컷 정상 생성, 코너 뷰 구도 양호.

## ② graphrag 백엔드 통합 (핵심)

graphrag 레포(`liminal-cipher/graphrag`)의 기능을 프론트 인룸 UI에 연결. 백엔드 LLM은 **OpenAI 키 탑재 후 정상 작동**.

### ✨ 의미부여 (mnemonic) — `POST {graphrag}/mnemonic`
- 핫스팟 사물 + 학습 개념을 **"어거지로" 연결**한 기억 장면(마크다운) 생성.
- 우상단 카드 "✨ 의미부여" 버튼 + 좌측 챗봇 "✨ 의미 매치" 칩(동일 기능) + 개념 선택 모달.
- 마크다운 렌더(`mdLite`: 헤더·표·목록·굵게). E2E 검증: "식탁 ↔ 사간원" 기억 장면 정상 생성.
- **저장·재표시**: 생성 결과를 계정별 localStorage에 저장(`mp_mnemo:{acct}:{palace}:{사물}:{개념}`) → 다시 열면
  그대로 표시 + "🔄 다시 생성". 클라우드 저장 best-effort POST(`/api/mnemo-save`) 훅 포함(백엔드 미구현, 아래 참고).

### 💬 인룸 RAG 챗봇 — `POST {graphrag}/query` · `/jobs/{id}/query`
- 도우미 자유질문을 graphrag RAG로 라우팅(`window.mpAssistantAsk`). 라이브 잡이면 `/jobs/{id}/query`, 아니면 쇼케이스 `/query`.
- 답변에서 `[Data:...]` 인용 토큰 정리 + **관련 개념 칩**(sources.entities→방 마커 매칭) 클릭=해당 학습 마커로 이동.
- E2E 검증: "조선 통치제도?" → 의정부·6조 정확 답변 + 📍세종·세조·정도전 칩.

### 🔬 정밀 객체 인식 (gpt-4.1 비전)
- 백엔드 신규 `POST /api/vision-label`(main.py): 크롭 이미지를 **멀티모달 LLM(gpt-4.1/4o 비전)** 에 보내
  한국어 사물명 반환(`{label}`). `llm_chat_config()` 재사용.
- "정밀" 버튼: **마커가 가리키는 대상을 크롭**해 vision-label 우선 호출(미설정 시 Azure Computer Vision `/api/detect` 폴백).
  대상이 프레임을 채워 인식 정확도↑. 라이브 검증: 침실 가구 크롭 → "소파" 인식.

### 📝 인룸 퀴즈
- 이 방 학습내용으로 퀴즈 생성→풀기→채점→**오답은 관련 마커로 복습**. 챗봇 "📝 퀴즈" 칩.
- 프론트는 `POST {graphrag}/quiz/json`(질문) + `POST {graphrag}/quiz/grade`(채점, 기존재)를 호출.

## 🔴 사용자 / 백엔드 액션 (남은 것)

### 1) graphrag에 `/quiz/json` 추가 (인룸 퀴즈 작동 조건)
`backend/quiz/quiz_page.py`에 아래 블록 추가(기존 `_store_quiz`·`_ANSWER_KEYS`·`EvidenceBuilder`·`generate_quizzes` 재사용):

```python
# ── JSON API (메모리 팰리스 프론트 연동) — HTML 라우트와 별개, 같은 generator 재사용 ──
_builders: dict[str, EvidenceBuilder] = {}

def _builder_for(snapshot: str | None) -> EvidenceBuilder:
    """스냅샷별 EvidenceBuilder 캐시. None/미존재면 korean_history(데모)로 폴백."""
    key = (snapshot or "korean_history").strip() or "korean_history"
    snap_dir = ROOT / "snapshots" / key
    if not snap_dir.exists():
        key, snap_dir = "korean_history", SNAPSHOT_DIR
    if key not in _builders:
        _builders[key] = EvidenceBuilder(snap_dir)
    return _builders[key]

class QuizJsonRequest(BaseModel):
    topic: str = ""
    count: int = 6
    quiz_types: list[str] | None = None
    snapshot: str | None = None   # 등록 스냅샷 키 또는 job_id

@router.post("/quiz/json")
async def quiz_json(req: QuizJsonRequest):
    """프론트(메모리 팰리스)용 JSON 퀴즈. 질문만 반환(정답 은닉, /quiz/grade로 채점)."""
    builder = _builder_for(req.snapshot)
    selected = builder.select_candidates(topic=req.topic, count=req.count)
    result = await generate_quizzes(selected, count=req.count, quiz_types=req.quiz_types or None, topic=req.topic)
    quizzes = result.get("quizzes") or []
    quiz_id = _store_quiz(quizzes) if quizzes else None
    questions = [{k: v for k, v in q.items() if k not in _ANSWER_KEYS} for q in quizzes]
    return {"quiz_id": quiz_id, "questions": questions, "mode": result.get("mode"), "warning": result.get("warning")}
```
- 쇼케이스(korean_history)는 즉시 작동. **라이브 PDF 퀴즈**는 `_builder_for`가 `var/jobs/{id}` 스냅샷도 찾도록 확장 필요
  (serve.py의 잡-스냅샷 경로 해석 재사용 권장).
