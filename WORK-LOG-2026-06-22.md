## 1. 무한 로딩(스튜디오) — 근본 원인 수정
- **원인**: `.rp-modal`/`.node-lock`에 `[hidden]` CSS 규칙이 없어 `display:flex`가 `hidden` 속성을 덮음 → 미리보기 모달("촬영 중")이 **항상 풀스크린으로 떠 있던** CSS 특이도 버그. (compose·memory-walk엔 이미 적용, 스튜디오만 누락)
- **수정**: `.rp-modal[hidden]`/`.node-lock[hidden]{display:none}` + 기본문구 중립화 + city 로드 실패 시 로딩표시 제거·탈출구 + 부트 fetch 타임아웃 + model-viewer 에러 핸들러.
- **교훈**: `modal.hidden`(속성) ≠ 화면 표시. 검증은 `getComputedStyle(el).display`로. → 메모리 `feedback_verify_rendered_state`.

## 2. 객체 인식 / 핫스팟 파이프라인 (memory-walk, 전 방 적용)
- **마커 중복·겹침**: 솎기가 3D 거리라 바닥물체+공중물체가 같은 바닥자리에 겹쳐도 안 걸러짐 → **바닥평면(XZ) 0.42m 겹침 솎기** 추가(카페·교실 의자 ≥0.5m는 보존).
- **허공/빈 바닥 마커**: 합성 '기억 지점'을 빈 바닥 격자 대신 **검출 가구 곁(중심쪽 0.6~0.9m)** 배치. 가구 0개일 때만 격자 폴백.
- **라벨 정확도**:
  - geomGuess: 넓은 선반을 '스탠드'로 오분류하던 것 → **폭·깊이로 책장/스탠드/수납장 구분**.
  - `box·basket·bottle·umbrella·kite·globe` 등 한글화.
- **비전 자동 명명(신규)**: 검출 직후 번호 마커 박힌 방 사진(4각도)을 GPT-4.1 비전(`/api/label-room`)에 보내 번호별 한글명으로 1차 교정 → 입장 전 동기, GLB별 캐싱, 실패 시 폴백(무회귀).
- **사물별 정밀 자동화(신규)**: 입장 후 배경에서 각 마커를 **오프스크린 close-up 렌더 → `/api/vision-label`**(사물이 프레임을 꽉 채워 가장 정확). 중앙 조준 십자선 + 프롬프트를 "중앙 사물만, 빈 곳이면 '없음'"으로 → 바닥 마커가 옆 큰 가구로 둔갑하던 것 방지(없음→'빈 자리'). precise 캐시로 재입장 시 재호출 없음.
- **이름 신뢰 하이브리드(핵심)**: 메쉬 이름이 실제 가구 단어면(label이 "obj숫자"/"spot"이 아님) **그 이름을 신뢰하고 비전 교정 제외**, 비전은 이름없는(geomGuess)·합성 지점에만. → living의 식탁→꽃병·소파→화분 오인 해결. **단 모호·오명명 이름(umbrella·kite·globe·object)은 신뢰 제외 → 비전이 판단**(우산→화분/커튼 교정).
- **동선 순서**: '각도 스윕'(좌우 횡단·지그재그) → **최근접 경로(거리+진행방향 유지)**. living 방 횡단 점프 3→1, 총경로 ~26→17.9m.
